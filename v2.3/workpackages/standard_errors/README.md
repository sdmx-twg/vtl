# Concept Note: Standard Errors
This work package seeks to define a set of codes for errors (and warnings) to be reaised by all engines in case of anomalies detected in the VTL process
(syntax errors, semantic errors, runtime errors).

## I. Overview and Purpose

The main goal of this workpackage is to standardise the behaviour of alla VTL engines in order to increase reliability and interoperability; the starting point 
will be the current list of errors described in the Technical Notes document.

## II. Scope and Approach

Here is a non-exhaustive list of the topics that would be addressed:

-	Integrate and the current error list with the ones contained in all VTL engines' implementation and uniform codes and descriptions
-	Possibly associate to all set of operators the list of possible errors
-	Review the specific behaviour for some operators (e.g. cast and stddev_samp) as agreed during last VTL Workshop in Paris
